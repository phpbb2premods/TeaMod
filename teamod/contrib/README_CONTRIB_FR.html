Mise en cache des templates

Auteurs: Crimsonbane, psoTFX and nathan

Mise à jour de la mise en cache des fichiers

Durant le développement de phpBB2, un problème récurrent de charge CPU s'est fait sentir. phpBB 2.0.0 a la particularité, par rapport à ses contemporains, d'effectuer de nombreuses tâches pendant la navigation des utilisateurs. Alors que cette particularité permet des upgrades et des modifications plus simples, cela mène aussi à une plus grande demande envers le serveur. Pour aider à atténuer ce problème, un système de mise en cache des templates a été créé, basé sur le travail de Crimsonbane. Ce système crée une version compilé de cache template, ce qui résulte en une baisse significative de la charge serveur, voire du temps de génération des pages.

Deux versions du sytème de cache sont présentes dans ce répertoire. L'une utilise des fichiers texte, l'autre une base de données MySL. Des teste ont montré que, en général, la version "texte" donne de meilleurs résultats. Ce pourrait néanmoins ne pas être votre cas. Libre à vous de tester les deux systèmes.

Dans les deux cas, il vous faut renommer votre actuel fichier includes/template.php et le remplacer par le template_*.php que vous souhaitez utiliser.

template_file_cache.php

To use the flat file system you need to create a directory called 'cache' in your phpBB root directory (it may already exist and contain some default files, do not delete these). This directory must be writeable by the web server, e.g. global write access. Under UNIX systems this is typically achieved by using chmod a+w cache or chmod 777 cache. Many FTP applications also allow you to change permission properties. See your applications manual for details on how to do this.

Please note that creating a world writeable directory is of course a potential security issue. Most of the time this is not a great issue, however on virtual hosting accounts there may be users who have nothing better to do than cause problems. Therefore you should monitor the status of this directory regularly and report any problems to your hosting provider (who will be in a position to take action against any offending user).

NB: The directory required for this version of the file caching template system differs from previous releases. You can safely delete the old caching directories.

template_db_cache.php

Pour utiliser le système "base de données" (notez qu'il n'est pour le moment compatible uniquement avec MySQL), il vous faut créer la table suivante (en remplaçant le préfixe par le votre si vous avez utilisé quelque chose de différent de "phpbb_"):

CREATE TABLE phpbb_template_cache (
  template_name char(255) NOT NULL default '',
  template_handle char(50) NOT NULL default '',
  template_cached int(11) NOT NULL default '0',
  template_echo tinyint(1) NOT NULL default '1',
  template_compile text NOT NULL,
  PRIMARY KEY  (template_name)
)

Ces fichiers ne devraient pas être introduits dans d'autres mods ou plugins. Ils ont été testés, mais nous ne pouvons garantir leur stabilité dans tous les cas. Ce système est basé sur le travail de Crimsonbane et bien sûr sur le système de template originel de Nathan.

Système de confirmation visuelle

Auteur: psoTFX

Malheureusement, il y a des gens "ici-bas" qui n'ont rien d'autre à faire que de causer des problèmes aux autres. Encore pire, certains groupes créent des applications qui causent des problèmes. Un façon très ennuyeuse (et absolument sans intérêt) de le faire est l'enregistrement en masse d'utilisateurs sur votre forum. Ce Mod, futur fonctionnalité de la 2.2 à venir, aide à éliminer ou réduire ce problème. Il génère une série aléatoire de 6 caractères alphanumériques sous forme d'images png. L'utilisateur doit alors rentrer ces caractères avant de pouvoir s'enregistrer. S'il échoue 3 trois fois de suite, l'utilisateur est bloqué pour le reste de la session.

Pour tirer le meilleur de ce Mod, vous devriez utiliser la version 4 de PHP, et avoir la librairie zlib installée et lancée. Il est assez courant que zlib soit installée par défaut. Si vous n'êtes pas sûr qu'elle le soit, demander à votre hébergeur ou regardez ce que donne un phpinfo();. Ce mod ne nécessite pas GD, Imagemagick, NetPBM ou tout autre librairie. Si zlib n'est pas disponible, le Mod utilisera une méthode plus simple de rendu. Néanmoins, cette méthode plus simple devrait toujours bloquer la majorité de vos fauteurs de trouble.

phpBB 2.0.9 contient certains morceaux de ce système préinstallés. Pour finir l'installation, vous décompressez le contenu de l'archive visual_confirmation.zip à la racine de votre répertoire phpBB. Ecrasez les fichiers déjà existants. Notez bien que si vous avez déjà installé des Mods, qui auraient modifié profile.php, includes/usercp_register.php, admin/admin_board.php ou templates/subSilver/admin/board_config_body.tpl vous perdrez ces modifications. Si vous êtes dans ce cas, nous vous conseillons de repérer les différences entre les fichiers et d'opérer les modifications manuellement.

Les packs de langue anglais et français contiennent déjà les chaînes nécessaires à ce mod. D'autres langages peuvent être implémentés : les chaînes sont listés ci-dessous? Prenez garde à ne pas traduire le %s présent dans l'une d'elle. C'est un repère pour un autre morceau de texte qui sera ajouté par phpBB.

Diagnostic Base de Données

Auteurs: Blade, AL, Techie-Michael, Black Fluffy Lion

dbinformer.php

NOTE: dbinformer.php fonctionne désormais avec touts les DBMS supportés par phpBB.

Un grand nombre de personnes ont rapporté des erreurs could not connect to database avec phpBB, spécialement lors de l'installation. Pour y remédier, ce script a été réalisé par l'équipe du support de phpBB. Pour l'utiliser, entrez le nom de votre serveur de base de données, le nom de la base, l'utilisateur, le mot de passe et le préfixe que vous voudriez que le script utilise (ex: phpbb_), et choisissez votre type de base de données. Précisez aussi si vous voulez générer un config.php à partir des données entrée, et cliquez sur Submit.

Ce script vous dira si la connection entre votre serveur et votre serveur de base de données a été établi. Si ce n'est pas le cas, il affichera un message d'erreur descriptif. Si la connexion s'est bien passé, il listera les tables de votre base de données; les tables commençant par le préfixe que vous avez choisi apparaîtront en gras. Si vous choisissez de générer un nouveau fiichier de configuration (utile si votre fichier a été accidentellement effacé, non envoyé,...), son code apparaîtra en base de la page. Copiez tout simplement ce code dans un éditeur de texte et enregistrez le en tant que config.php ou cliquez sur Download. Assurez vous que ce fichier n'a que 19 lignes, et uploadez le à la racine de votre forum phpBB.

Notez aussi que ce fichier, comme l'ensemble du répertoire contrib, devrait être effacé après utilisation. Ne laissez pas ce fichier en place si ce n'est pas nécessaire.
